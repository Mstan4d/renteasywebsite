The Core Concept

Every rental unit can be in one of three states:

  • Not managed by you → ignore
  • Managed by you but not on RentEasy → external property
  • Managed by you and listed on RentEasy → for marketing

Whether a unit is on RentEasy or not, once it's rented, you track it the same way in your system.

How Units Enter Your System

External Property (Manual Entry)

  1. You meet a landlord → They hire you to manage their property
  2. Add them as a LANDLORD in your system
  3. Add their PROPERTY (building/compound)
  4. Add UNITS (apartments) to that property
  5. Each unit is now in your system (vacant)
  6. When rented → Add TENANT to that unit
  7. Track payments for that unit forever

RentEasy Listing (Marketing First)

  1. You have a vacant unit (maybe not in your system yet)
  2. Post it on RentEasy to find a tenant
  3. It's just a LISTING (not yet a unit in your system)
  4. Someone rents it → Now you need to track it permanently
  5. CONVERT the listing to a UNIT
  6. System creates a PROPERTY and UNIT
  7. Add the TENANT → Track payments forever

The Four Core Entities

Landlords – who own properties
Properties – buildings/compounds that contain units
Units – individual apartments (source of truth)
Tenants – people living in units

And one temporary entity: Listings – units currently for rent on RentEasy.

Complete Lifecycle

Landlord Lifecycle

  • Estate firm meets a landlord → Adds to system (name, phone, email, bank details)
  • Add properties for this landlord
  • View all properties/units for this landlord in one place
  • When sending payments, bank details are ready

Property Lifecycle

  • Created by estate firm, linked to a landlord
  • Contains one or more units
  • Can be active or archived (when you stop managing it)
  • Archived only when ALL units are vacant

Unit Lifecycle (The Heart of the System)

  • Created: from external property or from RentEasy listing conversion
  • Vacant: can be posted to RentEasy (creates listing) or rented directly
  • Tenant added: search existing RentEasy user or add manually (name/phone). Unit becomes 'occupied'
  • Ongoing payments: record payment → PDF receipt → send to tenant (in-app or SMS)
  • Tenant moves out: remove tenant → unit becomes vacant → tenant saved to history
  • Stop managing property: only if all units vacant → archive property

Listing Lifecycle (Temporary)

  • Created when posting a vacant unit to RentEasy
  • Links to the unit (if unit exists) or standalone
  • Status: 'active' while looking for tenant
  • When rented: if linked → mark unit occupied, add tenant; if standalone → convert to unit first
  • Status becomes 'rented' (kept for history)
